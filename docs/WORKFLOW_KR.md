# Amicus 개발 워크플로우 (기본)

본 문서는 Amicus 레포에서 "모든 작업"에 기본적으로 적용되는 개발/검증/문서화 절차를 정의합니다.

## 목표

- 로컬에서 반드시 재현 가능한 상태로 검증한다.
- 작업 결과를 사람이 빠르게 검토할 수 있게 "시각화"(요약 리포트)한다.
- 기능 문서는 기본적으로 한국어로 작성한다.

## 기본 체크리스트

### 1) 작업 완료 후 로컬 검증

아래 명령이 로컬에서 성공해야 합니다.

```bash
bun run verify
```

`verify`는 다음을 순서대로 실행합니다.

- `bun run typecheck`
- `bun run build`
- `bun run test`

### 2) 작업 내용 시각화(리포트)

리포트는 변경점과 검증 결과를 Markdown으로 요약해, 리뷰어가 빠르게 확인할 수 있게 합니다.

```bash
bun run report
```

출력 파일:

- `reports/task-report.md`

리포트에 포함되는 항목(최소):

- 현재 브랜치/커밋
- 변경 파일 목록 및 diffstat
- 실행한 검증 커맨드 목록

### 3) 문서(기본 한국어) 업데이트

기능을 추가/수정했다면 문서를 반드시 함께 갱신합니다.

- 패키지/기능 설명 문서: 각 패키지의 `README.md` 또는 `docs/` 하위 문서
- 한국어를 기본으로 작성하고, 필요 시 영어 번역을 추가합니다.

## 참고

- CI(GitHub Actions)는 `bun run verify`가 통과하는지를 기준으로 합니다.
- 테스트는 루트에서 의도치 않은 외부 디렉토리(예: `next.js/`)를 타지 않도록, 패키지 단위로 실행합니다.
